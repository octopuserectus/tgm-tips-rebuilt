Sliding over holes (窪みを越える - kubomi o koeru)

A basic example of using IRS to land with the 2-wide side down to cross a 1-cell gap.
It might be good to get used to spawning J and L pieces in this orientation.
You can also cross gaps by using steps.
"J-piece 3 Rotations" (青の３回転 - ao no 3 kaiten):
Prevent falling into the gap by orienting the wide side downwards.
In most cases, IRS is needed for the first rotation.
The pattern in the right diagram is slightly more difficult and requires performing the first two rotations quickly.
Learning the feel of intentionally delaying the DAS charge (溜め遅れる - tame okureru) is useful.

Examples Where IRS Should Not Be Used

S and L pieces are easy shapes to move left, but when placing them on the 4th column, you must not use IRS.
Don't wait until "that moment" comes; become able to recognize "No IRS if S or L comes" as soon as the setup is formed.
Both involve the operation "left, left, rotate".
If you hesitate, the block will lock, so become able to input quickly and confidently without visually tracking the block's movement.

Moving the I-piece to the Edge

Placing the I-piece on the right edge requires a 2x2 space.
Therefore, an I-piece can be placed on the right edge in a stair-like stack shape.
As long as a 2x2 space is secured up to the edge, other bumps don't matter.
Be able to confidently throw it to the right.
On the other hand, placing it on the left edge requires a 3x2 space.
Therefore, an I-piece cannot fit into the left stair shape.
Examples of the operation "horizontal, horizontal charge, rotate". The key is quickly inputting sideways twice and holding the second input.
Especially, it's easier if the rotation causes a drop (lock delay reset), but severe if not. It's too late if you wait to visually confirm the block's appearance, so judging the block appearance timing is necessary.
So, what can be done when the shape is like this?
Comparing the 20G rule "if there's a step, it always falls to the lowest point" and the I-piece's rotation rule, it seems like it would get caught on the 3rd column.
However, in this case, if you

"DAS charge left with the piece flat all the way to the edge, then rotate"

you can place it in the 1st column.
The principle is explained below.

Synchronous Actions (シンクロ - Synchro)

Performing multiple operations within a single frame.
As mentioned in Chapter 1, a frame is the smallest unit of screen drawing, but within each frame, it's known that judgments are performed in the order:
Rotation → Horizontal Movement → Drop

If both rotation and horizontal movement are performed in one frame, horizontal movement becomes possible even in mid-air during 20G.
Generally, "Synchro" refers to advanced techniques like those introduced in Chapter 12, but here it's broadly considered as a "phenomenon" for explanation.

↓

Let's look at the example above.
From this state, press the button while holding left charged.
As mentioned in Chapter 1, DAS charge is the same as inputting sideways every frame, so "sideways and rotation are input simultaneously".
Then...

↓

First, the rotation judgment occurs (rotates if possible).
This happens internally in the computer and is not drawn on screen.

↓

Next, the horizontal movement judgment occurs (moves horizontally if possible).
Still not drawn on screen here.

Furthermore, after that, the drop judgment occurs (drops to the bottom in 20G), and the result is drawn on screen.
Since it's in DAS charge state, it drops into the 1st column in the next frame.

Of course, if you rotate after reaching the left edge with the lever neutral, it stands up in the 3rd column from the left as usual, so use appropriately depending on the situation.

Reverse Somersault (逆上がり - saka-agari)

This also seems like it would get stuck in the 3rd column, but it can be easily propped up using the operation "left charge, rotate".
A slightly difficult pattern: move with right charge, then re-charge left.
Generally, this might be what's referred to as a reverse somersault (逆上がり - saka-agari).
Be careful, as insufficient charge will cause it to slide down, and rotating too late will cause it to lock as is.

By the way, if you can input horizontal and rotation simultaneously, charging isn't necessary. It's good to try challenging it on terrain where sliding down doesn't matter.

Valley Crossing (谷越え - tani-goe)

Without using advanced 3-rotation passes like those introduced in Chapter 9, it's sometimes possible to cross a 2-column wide hole using steps.
This is often used in conjunction with the reverse somersault introduced above.
Assess the stack shape and become able to do it confidently.
